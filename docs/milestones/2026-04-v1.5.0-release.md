                     ↑ 覆盖（栈级优先）
┌─────────────────────────────────────────────────┐
│  Generic Layer                                   │
│    assets/pl/skills/   (v1.5 起有内容)          │

## 6. Breaking Changes

### ⚠ Minor Breaking（3 项）

1. **build-verification rule 的 gradle 命令外化**
   - 原 rule 直接写 `./gradlew compileKotlin`
   - 新 rule 改为从 `.pl-adapter.yaml` 的 `build_adapter.commands` 读取
   - 影响：直接复制 rule 不经 adapter 装配的用户需建立 `.pl-adapter.yaml`

2. **文件名归一化**
   - `git-commit-rules.md` → `git-commit.md`
   - `serialization-rules.md` → `kotlinx-serialization.md`
   - `acceptance-criteria-rules.md` → `acceptance-criteria.md`
   - `build-verification-rules.md` → `build-verification.md`

3. **pipeline-master agent 六阶段 → 七阶段**
   - 新增 SMOKE 阶段（介于 VERIFY 和 OBSERVE 之间）
   - 影响：自定义 `.state.md` 若硬编码阶段枚举需兼容

详见 [`docs/migration/v1.4-to-v1.5.md`](../migration/v1.4-to-v1.5.md)。
