## 一、本阶段主线叙事

> **从"事件可观测"到"事件可被看见"，再到"实时被看见"。**

v1.2.0 做完"可执行契约层"之后，pl/piao 本体已经具备机器可求值的门禁能力。但下一个瓶颈浮现：

- **所有事件都结构化写进 jsonl，却没人能直接看**
- 观察者必须 `tail -f` + `jq` 手工解析，信号强噪音大
- 做 retro / 封版时要翻日志才能讲清楚"第 k 次跑出了什么"

v1.3.x 系列就是回答这个问题的三步：

| 子版本 | 问题 | 答案 |
|--------|------|------|
| **v1.3.0-alpha** | 事件存在但人看不到 | Observe 层：event-v1.3 schema + fs watcher + 业务语义规则引擎 + Dashboard 双视图（静态） |
| **v1.4.0-alpha** | 事件多了但看不出模式 | retro-miner：离线挖掘频次 / 共现 / 反模式 / 异常（命令行 json 产出，10 条合成验证通过） |
| **v1.3.2-alpha.2** | 事件在动但看板不动 | Dashboard live reload：自写 SSE server + EventSource 客户端 + 降级 + user guide |

三者互补：observe 是**入库**，retro-miner 是**离线分析**，live-reload 是**实时消费**。

## 三、关键技术决策与理由

### 3.1 为什么不用 MCP / WebSocket / fswatch

| 被否决方案 | 否决理由 |
|-----------|---------|
| MCP | 配置复杂，RPC 契约会把后续演化卡死 |
| WebSocket | 单向推送用不着握手 + 额外库 |
| fswatch / inotifywait | 引入 OS 依赖，Mac/Linux 行为不一致 |
| npm/pip 三方库 | 破坏"零依赖"原则 |

### 3.2 为什么选 SSE + 自写 HTTP server

- **SSE**：浏览器 `EventSource` 原生自动重连，客户端代码极简
- **自写 server**：Python stdlib `http.server` 不支持长连接端点；自己继承 `BaseHTTPRequestHandler` 只多 200 行，保持零第三方依赖

### 3.3 为什么放弃 meta-agent / 双轨设计

过程中讨论过两个更复杂方案，都被否决：

| 方案 | 否决理由 |
|------|---------|
| meta-agent 自动生成规则 | 质量不足，且 rule/skill 本身就是 memory，不需要额外一层 |
| 软硬双轨（LLM 主动触发 + fs 被动观察） | 不确定性叠加，难以追责；单硬轨（fs watcher）已够 |

### 3.4 为什么选 B 方案（先 retro-miner 再定 E5）

E5 trace-verifier 最初设计是"7 条规则，其中 5 条 error"，过于严格会重蹈 retro-v1 的覆辙。
选择 B：**先让数据说话**，积累足够 pattern 再定契约，避免"设计先行"的老毛病。

## 四、过程中踩到的坑（按严重度排序）

### 严重：v1.3.2-alpha 的 snapshot 丢失 bug

- **根因**：`tail_jsonl_watcher` 的 `snapshot_sent` 变量是 watcher 全局状态，不是 per-subscriber。第二个订阅者错过初始全量 → 只看到空白。
- **触发路径**：同一个 change 开多个 tab、或者 SSE 重连、或者 Playwright 测试时
- **修复（v1.3.2-alpha.2）**：Broadcaster 引入 `snapshot_fn`，订阅时对每个新订阅者独立生成当前全量
- **教训**：共享资源里的"一次性事件"必须显式考虑多订阅者语义

### 中：CHANGELOG 替换粒度过粗

- 现象：用 replace_in_file 插入新 section 时误把 v1.2.0 标题替换成 v1.3.0-alpha，导致整段错位
- 教训：**每次 replace 后用 `grep '^## \['` 扫一眼章节结构**作为 self-check

### 低：其它 4 个（环境类）
- Shell 变量展开顺序：PL_HOME 必须 `export` + 引号
- 端口残留：老 python 进程 `pkill -9` 才能清掉
- Playwright MCP 冷启动超时：备用路径 curl + headless chrome
- Playwright screenshot 缓存错位：文件本身对，图片预览缓存错

完整汇总见 [`../retros/evidence-2026-04-v1.3.2/README.md`](../retros/evidence-2026-04-v1.3.2/README.md)。
