# spec.md · blog-post-editor

> **场景**：博客系统的"新建/编辑文章"页面（脱敏示例）
> **change-id**: `feat-post-editor`
> **生成于**: 本示例用于 `spec-normalizer@1.0.0`

本示例特意选了一个**和电商完全不同的领域**，用来展示 spec-normalizer 的通用性。

---

## 1. 基本信息

| 字段 | 值 |
|------|---|
| id | `post-editor` |
| 业务域 | 内容创作 / Content |
| 入口 | 侧边栏 → "写文章"，或文章详情 → "编辑" |
| 复杂度 | 高（富文本、媒体上传、自动保存，含实时协作 TBD） |

## 2. 功能清单

| ID | 优先级 | 描述 | 验收标准 |
|----|--------|------|---------|
| R1 | P0 | Markdown 编辑器（含预览） | 键入 → 预览区 1s 内刷新 |
| R2 | P0 | 标题/标签/分类（元数据区） | 必填校验：标题不为空 |
| R3 | P0 | 插入图片（本地上传 / 粘贴 URL） | 上传进度可见、失败可重试 |
| R4 | P0 | 保存草稿 | 自动保存（离最后一次键入 3s 后） |
| R5 | P1 | 发布（draft → published） | 发布前二次确认，检查 meta 完整 |
| R6 | P1 | 历史版本查看 | 列出最近 10 个 draft 快照 |
| R7 | P2 | 实时协作（多人同时编辑） | **(deferred to v2)** |

## 4. 数据依赖

| 接口 | 方法 | 路径 | 说明 | P0? |
|------|------|------|------|-----|
| 创建 draft | POST | `/api/posts/draft` | R4 首次触发 | ✅ |
| 自动保存 | PUT | `/api/posts/{id}/draft` | R4 周期性 | ✅ |
| 上传图片 | POST | `/api/upload` (multipart) | R3 | ✅ |
| 发布 | POST | `/api/posts/{id}/publish` | R5 | |
| 版本列表 | GET | `/api/posts/{id}/revisions` | R6 | |

**本地缓存**：对 R4 的 draft 内容，前端用 IndexedDB 做离线 fallback（NF1）。

## 5. 状态机

```
empty ──▶ editing ──auto-save──▶ saving ──▶ saved
           │   │                    │
           │   └─(失败)──▶ save_error ──(retry)──┘
           │
           └─(publish 按钮)─▶ publish_pending ──(确认)──▶ published
```

## 6. 跳转关系

**入口**：
| 来源 | 参数 |
|------|------|
| 侧边栏 "写文章" | `?mode=new` |
| 文章详情 "编辑" | `?mode=edit&id=<post_id>` |

**出口**：
| 目标 | 触发 |
|------|------|
| 文章详情页 | R5 发布成功后 |
| 列表页 | 用户关闭（有未保存草稿时先弹确认） |

## 7. 埋点

- PV: `editor_open` (params: mode)
- 核心事件：`editor_autosave_success` / `editor_autosave_fail` / `editor_publish`
- 非功能：编辑响应时间 P99、自动保存失败率

## 9. 验收基线

- [ ] 所有 P0 功能完成
