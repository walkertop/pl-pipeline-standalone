# StructuredSpec: [变更/页面名称]

> **模板版本**: v1.1 (pl-pipeline)
> **用途**: 将多形态需求输入（PRD/Figma/截图/文字）标准化为机器可读的结构化规格
> **消费者**: Planner Agent (PLAN 阶段)、Coder Agent、Guardian Agent
> **门禁**: 所有 BLOCKING 问题解决后方可进入 PLAN 任务拆解

---

## 1. 变更概述

| 字段 | 值 |
|------|-----|
| Change ID | `change-id` (kebab-case，如 `add-user-login` / `migrate-checkout`) |
| 变更类型 | 页面迁移 / 功能新增 / 架构重构 / 性能优化 / Bug 修复 |
| 关联页面 ID | `page_id`（若涉及页面，对应技术栈的页面注册名） |
| 业务域 | 由项目/adapter 定义（如 交易 / 会员 / 活动 / 用户 / 营销 / 基础设施） |
| 用户入口 | 从哪里进入此页面/功能（如：订单列表 → 点击卡片） |
| 对标旧实现 | 旧代码源路径（如 `<legacy-source>/<path>`）或 `无` |
| 复杂度预估 | 🔴高 / 🟡中 / 🟢低 |
| 预估组件数 | 数字（基于 Screen Map 估算） |

## 3. UI 结构描述 (Screen Map)

> 用 ASCII 线框图描述页面布局，每个区域标注对应的功能 ID 和预期使用的组件。

```
┌─────────────────────────────┐
│        NavBar               │  ← 复用: component.common.NavBar
├─────────────────────────────┤
│  [F1] 功能区域 1             │  ← 新建/复用: XxxComponent
│  [描述展示内容和交互]         │
├─────────────────────────────┤
│  [F2] 功能区域 2             │  ← 复用: DJCList<T>
├─────────────────────────────┤
│  [F3] 功能区域 3             │  ← 新建: XxxComponent
├─────────────────────────────┤
│        底部安全区             │
├─────────────────────────────┤
│  [F4] 底部操作栏             │  ← 复用: DJCButton
└─────────────────────────────┘
```

> **标注规范**:
> - `复用: 组件名` — 在 ARCHITECTURE_SNAPSHOT 中能找到的现有组件
> - `新建: 组件名` — 需要新开发的组件
> - `待定: 描述` — 尚未确定实现方式

## 8. 不确定项 (Open Questions)

| ID | 问题描述 | 阻塞等级 | 建议处理 | 找谁确认 | 状态 |
|----|---------|---------|---------|---------|------|
| Q1 | [问题描述] | BLOCKING | [建议方案] | @产品 / @设计 / @后端 | 待确认 |
| Q2 | [问题描述] | NON_BLOCKING | [当前假设: xxx] | @xxx | 按假设推进 |

> **阻塞等级定义**:
> - **BLOCKING**: 不解决则无法继续规划/开发，必须暂停等待确认
> - **NON_BLOCKING**: 可按假设推进，后续调整成本低

## 9. 验收基线 (Acceptance Baseline)

### 功能验收
- [ ] P0 功能全部实现: F1 ~ Fn
- [ ] P1 功能全部实现: Fn+1 ~ Fm
- [ ] 所有状态机路径覆盖

### UI 验收
- [ ] 与设计稿/旧版截图对比差异率 ≤ 5%
- [ ] 适配主流屏幕尺寸

### 数据验收
- [ ] 所有接口字段映射正确
- [ ] 边界处理: 空数据、网络异常、登录失效

### 性能验收
- [ ] 页面加载时间 < 1s（WiFi 环境）
- [ ] 列表滚动 60fps
